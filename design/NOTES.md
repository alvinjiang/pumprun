### Build & Deploy

```bash
# Client
python3 build.py                          # generates web/index.html (verifies JS syntax)
cd web && fyra push --non-interactive --appname pumprun  # deploys to pumprun.apps.fyra.sh

# Server  
cd server && go build -o pumprun-server . # rebuild binary
# Then commit + push to GitHub. User deploys manually to /opt/pumprun/
```

### JS template structure (in build.py)
The JS is an f-string `f"""..."""` with `{{` for literal braces. Data is embedded via Python variables:
- `{btc_data}` — BTC prices as JSON array
- `{basket_b64}` — Float32Array base64
- `{coin_data}` — coin index

### Key API endpoints (client calls)
- `GET API + '/pump/ledger'` — lobby stats  
- `GET API + '/pump/recent?n=20'` — ticker tape (also `?n=` for recent feed)
- `POST API + '/pump/run'` — record game (with `X-Api-Key` header)

API base set via `<meta name="api-base" content="https://api.gridrun.net">`
API key set via `<meta name="api-key" content="immadegen">`

### Server config
- Binary: `/opt/pumprun/pumprun-server`
- Flags: `-addr :8090 -data /opt/pumprun -allow-origin "https://pumprun.apps.fyra.sh,https://pumprun.fyra.sh" -api-key immadegen`
- Behind haproxy with `path_beg /pump/` ACL

### Data stored per game (db.jsonl)
```jsonl
{"key":"run:<uuid>","value":{"i":"<uuid>","s":1005,"p":50,"r":1.9,"t":[0,17,125,283,311],"d":175923}}
```
Fields: i=UUID, s=start day, p=banana percentile, r=return vs badger, t=trade days, d=dollar diff

### Game mechanics
- 365-day window, $10,000 start
- BTC quantity static during HODL, compounds with basket returns during YOLO
- Chart: 3 lines (green=ALTS, red=BTC, yellow=YOU), 600px, starts ±25%
- Verdict: HODLER/DEGEN/LUCKY APE/HONEY BADGER WINS
- Banana test: 1,000 permutations, chunked (50/tick), runs async after game end
- `p` field in POST uses banana test result (hardcoded 50 if test fails/not run)

### UX quirks to preserve
- Countdown defaults to YOLO (FOMO), free toggling during countdown
- Button colors: green=HODL, red=YOLO (opposite of what you'd expect)
- No Q key handler — summary only on game finish
- Summary card 9:16 portrait, 45% overlay opacity, aligned to chart top
- Progress bar 6px with gradient fill (#5a6080→#9a9fc0)
- Ticker tape full-width, fixed at top

### Design decisions
- Cryptowat.ch dark terminal aesthetic (pure black, SF Mono font, plain zero)
- Colors sampled from cryptowatch-big.png via vision model
- Summary modal styled as DeFi mobile wallet receipt
- Single HTML file, no framework, no dependencies
- Append-only JSON-lines server storage
- Short field names to save disk space (i,s,p,r,t,d)
- Comma-separated -allow-origin for multiple CORS origins
- `/pump/` path prefix to avoid haproxy conflicts with other services
